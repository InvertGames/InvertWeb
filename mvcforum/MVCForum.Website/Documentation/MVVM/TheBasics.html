The Basics

Create A Project

You need to create a new project to begin this journey.

Navigate to the project window and create an empty folder.

From here right click on the folder and choose uFrame->New Project

This will create a new project repository for you. The project repository will hold references to all of your graphs.

Create a new Subsystem Graph with the name 'Basics'

Graphs are used to create various sections of your project. Graphs can also share nodes between each other as long as they belong to the same project.

Create a 'Element' node with the name 'Player'

To create a node you need to right click on an empty space on the graph.

Then choose Add Element


Element

Create a 'View' node with the name 'PlayerView'

To create a node you need to right click on an empty space on the graph.

Then choose Add View


Element Views

Create a connection from Player to PlayerView's Element input slot.

Creating this connection means that the view will visually present the data on the player to the user in the 3d world.

You can always create multiple views to seperate different presentations of the same data. For instance, you could also create a PlayerUIView, which deals strictly with showing inventory, stats,...etc


Create a property with the name 'Name' on the 'Player' node

To create a node you need to right click on an empty space on the graph.

Then choose Add PropertiesChildItem


Create a command with the name 'Reset' on the 'Player' node

To create a node you need to right click on an empty space on the graph.

Then choose Add CommandsChildItem


Create a binding with the name 'Name Changed' on the 'PlayerView' node

To create a node you need to right click on an empty space on the graph.

Then choose Add BindingsReference


Save & Compile the project.

Saving and compiling is the process of taking the information you've created by graphs and putting it into code form. The generated code consists of both designer files (always regenerated), and editable files (only generated if it doesnt exist so it doesn't destroy your implementations).
