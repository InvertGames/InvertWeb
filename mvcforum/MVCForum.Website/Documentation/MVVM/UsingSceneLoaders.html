Using Scene Loaders

The purpose of this tutorial is to teach you the high level aspects of using uFrame, this includes SubSystems, and Scenes


Create A Project

You need to create a new project to begin this journey.

Navigate to the project window and create an empty folder.

From here right click on the folder and choose uFrame->New Project

This will create a new project repository for you. The project repository will hold references to all of your graphs.

Create a new MVVM Graph with the name 'ANYTHING'

Graphs are used to create various sections of your project. Graphs can also share nodes between each other as long as they belong to the same project.

Create a 'Scene Type' node with the name 'SceneB'

To create a node you need to right click on an empty space on the graph.

Then choose Add SceneType


Scene Types

Create a 'Element' node with the name 'Game'

To create a node you need to right click on an empty space on the graph.

Then choose Add Element


Element

Create a Command with the name 'LoadB'

To create a node you need to right click on an empty space on the graph.

Then choose Add CommandsChildItem


Now we need to add a command to load scene B.

Create a Command with the name 'UnLoadB'

To create a node you need to right click on an empty space on the graph.

Then choose Add CommandsChildItem


Now we need to add a command to un-load scene B.

Create a 'View' node with the name 'GameView'

To create a node you need to right click on an empty space on the graph.

Then choose Add View

Save & Compile the project.

Saving and compiling is the process of taking the information you've created by graphs and putting it into code form. The generated code consists of both designer files (always regenerated), and editable files (only generated if it doesnt exist so it doesn't destroy your implementations).

Open The Scene

Now add the code to the game controller to load and unload Scene B.

Right-Click on the Node and choose Open->Controller.cs

Now add the view to the scene.

Create an empty gameObject underneath the _SceneARoot game object. When creating scene types, everything should be a descendent of this root game object, this allows them to be destroyed by uFrame when needed.

On this empty game object click 'Add Component' in the inspector. Then add the 'view' component to it.

Now run the game from SceneA, select the gameobject and hit the LoadB button and UnLoadB button from the GameView's inspector.
