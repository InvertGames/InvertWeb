Dependency Injection
Getting Started > General uFrame Architecture > Dependency Injection

Dependency injection is a core concept in uFrame.  It decouples the various parts of a project from being solely "dependent" on others. Classes are not directly instantiated but "Resolved" by the dependency container.  This allows a class to only rely on a simple interface or abstract class rather than any concrete implementation it might need.

 

In uFrame there is a custom class called "uFrameContainer" that is the dependency container.  This class stores type mappings and even instances for a type.

Within SceneManagers there is a wrapper Property called "Container" for cleaner encapsulated access.  A scene manager is the first thing loaded by the GameManager, and it's setup method is the first thing to be invoked.  When it is invoked it has the responsibility have registering and dependencies that will be used in the context of the scene that the scene manager belongs to.  By default uFrame generates this method for you based on your subsystems and their Registered Instances.  However understanding how it works can greatly decouple your application.  This also allows you to manually work within uFrames core to access some of its powerful features.  To learn more visit the topics the below list.
