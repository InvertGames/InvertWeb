Computed Properties
Getting Started > Element Graphs > Computed Properties

Computed properties allow you to have calculate another properties value when any of its dependants change.  When a View-Model is constructed it will subscribe to any other properties that the computed is dependant on.  When their values change the computed property is recalculated and set.  This allows for a very declaritive way of describing logic.  Computed Properties are just like regular properties with some additional functionality. 

Computeds can be bound to just like any other property.

When you create a computed property node all the dependant properties should be connected to the computed's input.  You can also use computed properties as inputs to other computeds.  This gives you a lot of different options for other subscriptions as well.

 

Computed Properties are constructed in the View-Model's bind method.  There is a generated Reset{ComputedProperty} method that is invoked which sets up the computed property.  This looks like the following.

IsEmpty_ComputedProperty

    public virtual void ResetIsEmpty()
    {
        if (_IsEmptyDisposable != null) _IsEmptyDisposable.Dispose();
        _IsEmptyDisposable = _IsEmptyProperty.ToComputed(ComputeIsEmpty, this.GetIsEmptyDependents().ToArray()).DisposeWith(this);
    }
